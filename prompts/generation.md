# Hiring-pack generation — initial template

Generate artefacts only from a human-approved `RoleSpecification`. Preserve the distinction between must-have, preferred, and optional requirements. Do not add unsupported requirements or invent ZURU facts, benefits, policies, locations, or employment details.

Maintain traceability: every generated criterion and screening question must reference the approved requirement IDs that support it. Return structured data matching the supplied schema and surface any missing inputs rather than guessing.
